## 问题描述

### 当前系统的缺陷

现有平衡系统只进行**几何约束检查**：
- ✅ 质心X坐标是否在支撑范围内 `[min_x, max_x]`
- ❌ **不考虑力的平衡** — 导致两类问题

### 现实场景对比（用户图示）

#### 图1：质心超出但可稳住 ✓
```
状态: 左手拉 + 左脚蹬（对抗力）
结果: 尽管质心超出投影，因为有力矩平衡，所以稳住了
原因: 手和脚的力相互平衡，绕质心的扭矩≈0
```

#### 图2：质心在内但会开门 ✗
```
状态: 手脚配置不当，力方向不平衡
结果: 质心在支撑范围，但会导致"开门"（躯干旋转）掉落
原因: 力矩不平衡，躯干旋转力矩未被抵消
```

## 解决方案

### 架构设计

#### 第1层：力向量计算

**核心思想**：每个接触点（手、脚）都产生一个力向量

**手臂拉力**：
- 方向来自 IK 的肘→腕 向量（归一化）
- 大小为玩家出力（可简化为常数，或与肌肉激活度挂钩）
- 这个力产生的水平分量和竖直分量都会影响身体平衡

**脚的反作用力**：
- 脚踩岩点时，岩点对脚产生支持力
- 方向大致沿着大腿方向（髋→脚），因为腿是连接体
- 反作用力指向相反（牛顿第三定律）
- 脚离质心越远，同样的力产生的力矩越大

**思路**：不需要精确计算每个力的大小，可以：
- 手臂拉力：用常数代替（假设玩家一直全力拉）
- 脚反应力：用常数代替（假设脚提供的支持力固定）
- 关键是方向的计算准确

#### 第2层：力矩平衡检测

**核心概念**：选择质心作为旋转中心，计算所有力绕质心的力矩

**力矩计算原理**：
- 力矩 = 力臂 × 力的大小
- 在2D中，用向量叉积表示：`r × F = r.x * F.y - r.y * F.x`
- 这个标量值表示力绕质心的旋转趋势（正=逆时针，负=顺时针）

**为什么用质心**：
- 质心是整个身体的"平衡中心"
- 如果绕质心的力矩为零，身体不会旋转
- 这是物理学中的标准做法

**具体过程**：
1. 遍历所有吸附的手脚
2. 对每个吸附点，计算：力的作用点相对于质心的位置向量 × 该点的力向量
3. 累加所有力矩
4. 判断总力矩是否足够小（在容忍度内）

#### 第3层：改进的失衡判定

**改变逻辑**：从"质心在支撑范围"改为"质心在支撑范围 AND 力矩平衡"

**两个条件并行检查**：
1. **几何条件**（原有）：质心X坐标是否在支撑范围内
2. **力学条件**（新增）：绕质心的力矩是否接近零

**失衡判定**：只要其中一个条件失败，就开始累计失衡时间。两个条件都满足时才能恢复稳定状态。

**容忍机制**：允许短暂的失衡（通过失衡容忍时间）。如果超过容忍时间还没有恢复，才掉落。

## 实现步骤

### Phase 1: 力向量计算

**目标**：从现有 IK 数据中提取手臂拉力和脚反作用力

**手臂拉力获取思路**：
- 手臂本身就在拉扯，从 IK 的肘→腕 向量直接得到拉力方向
- 可以使用现成的 `BoneChain` 数据（肘位置、腕位置已有）
- 拉力大小设为常数或与手臂张力（绳长差）相关
- 需要在 Player 中添加方法获取当前两只手的拉力向量

**脚反作用力获取思路**：
- 脚踩的反作用力沿着大腿方向（因为腿连接了脚和躯干）
- 从 IK 数据中获取髋→脚 的方向
- 反作用力指向相反（推向身体），大小也设为常数
- 只有吸附的脚才有反作用力

**关键**：这两个力都是"虚拟力"，只用于判定平衡，不影响实际运动。可以简化处理，不需要精确物理模拟。

### Phase 2: 力矩计算

**目标**：计算绕质心的总力矩

**实现思路**：
- 质心位置从现有的 `getCenterOfMass()` 获取
- 对每个吸附的接触点（左右手、左右脚），计算：`(接触点 - 质心) × 力向量`
- 在2D中，向量叉积就是一个数值（用简单的叉积公式）
- 累加所有的力矩得到总力矩

**需要判断的情况**：
- 手吸附：使用手的拉力向量
- 脚吸附：使用脚的反作用力向量
- 脚着地（地面接触）：地面反作用力方向向上，大小任意（只要知道方向即可平衡判定）

### Phase 3: 平衡检测修改

**目标**：改进现有 `checkBalance()` 方法

**修改思路**：
1. 保留原有的质心几何检查（质心X在支撑范围内）
2. 新增力矩平衡检查（总力矩的绝对值 < 容忍度）
3. 两个条件都满足才认为稳定，任一失败则开始累计失衡时间
4. 超过失衡容忍时间才掉落

**核心改变**：从单条件判定变为双条件判定

### Phase 4: 调试与优化

**思路**：
- 参数调整：容忍度的设置（太大=>容易稳住，太小=>容易掉落）
- 可视化：在画面上绘制力矩信息，便于观察
- 测试场景：用各种手脚配置验证是否符合物理直觉

## 关键参数说明

**力的大小参数**：
- 基础拉力（手臂）：假设玩家一直全力拉，拉力大小为常数。实际值可在测试中调整。
- 脚反应力：脚踩岩点时提供的反作用力，也设为常数。
- 地面反应力：脚站在地面时的支持力，方向竖直向上。

**力矩容忍度**：
- 表示允许多大的力矩不平衡。太大则容易稳住（即使力矩很大），太小则容易掉落。
- 这是可调参数，需要在游戏测试中根据手感来确定。
- 建议初值不要太大，让力矩平衡的约束有一定意义。

**力矩衰减**：
- 当条件满足时，失衡时间应该快速恢复（衰减）。
- 这样可以避免瞬间的力矩波动导致掉落。

- 手臂拉力模型由固定值改为可变：

	- 根据拉力方向与岩点基准受力方向的夹角引入温和角度增益（最大约 +20%），以反映侧向分力，但避免过度放大。
- 支撑宽度会影响力矩容忍度：当支撑区（双脚或手脚形成的区间）较宽时，按比例放大 torqueTolerance，使宽站姿更稳定。若双脚均在地面/吸附，则进一步乘以 groundStabilityMultiplier（默认 2）。
- 为避免误判，引入“恢复性力矩”概念：当总力矩的方向能够抵消质心的几何偏移（即力矩方向与 -comOffset 同号）时，把该分量视为恢复力并抵消几何不稳定度（乘以 restorativeWeight，提高其影响力）。

**关于计时和掉落策略的决定（按要求保留原始行为）**
- 最初我尝试通过改变失衡计时增长速度和添加渐进恢复来获得更平滑的体验，但你要求“不要改变稳定时间掉落，也不要增加不稳定时间速度”。
- 已按要求恢复为原始掉落判定：
	- 失衡开始时以原始速率累加失衡计时器（每秒累加 1000 毫秒的等效值）；
	- 如果恢复到稳定（effectiveInstability <= threshold），则直接将计时器重置为 0（与最初实现一致）；
	- 不再对累加速率进行放大或按不稳定程度加速。

**为什么保留原始计时**
- 统一的容忍时间（balanceToleranceTime）是游戏体验的重要节奏参数：改变累加速率会影响玩家操作的容错空间和游戏手感。
- 因此在引入基于力矩的判定时保留原始时间机制，可以更安全地调整力学判定本身而不改变玩法节奏。

**后续建议**
- 如需更精细的调节，建议将以下参数暴露到 Inspector：`handPullForce`、`footReactionForce`、`torqueTolerance`、`groundStabilityMultiplier`、`restorativeWeight`、`instabilityThreshold`。这样可以在运行时微调手感而无需修改代码。
- 若希望玩家能主动控制手部用力（而非自动估算），可以增加一个“主动出力”输入（按键/长按），将手的输出与玩家输入绑定，能显著改善交互感受。
